# Business Rules — Preise (price disclosure by the voice agent)

Rules governing whether and how Kiki may state prices on calls: the Preisauskunft toggle, the auto-generated price-list knowledge base, and KVA draft pricing. Verified in code on branch `improvements-kiki-zentral` (2026-06-11).

## PRE-01 — Preisauskunft ist standardmäßig AUS: Kiki nennt am Telefon KEINE Preise (auch keine ungefähren) und bietet stattdessen einen unverbindlichen Kostenvoranschlag an.
- **Enforced by:** `backend/app/services/agent_config.py:render_price_info_block` (OFF branch renders the no-prices instruction into the `# Preise` block via `{{KZ_PRICE_INFO}}`)
- **Surfaced in UI:** Kiki-Zentrale → Preisauskunft (`frontend/src/components/kiki/ConfigSections.tsx:PreisauskunftSection`, "Standardmäßig deaktiviert")
- **Covered by test:** `backend/tests/agent_evals/scenarios.json::price_question_toggle_off` (LLM-judge eval, not pytest); no pytest unit test found for `render_price_info_block`
- **Prompt block:** `# Preise` ({{KZ_PRICE_INFO}}) in `agent_prompt_template.txt`
- **Status:** prompt-only (instruction is generated by backend code, but compliance happens in the LLM)

## PRE-02 — Bei aktivierter Preisauskunft nennt Kiki AUSSCHLIESSLICH Preise aus dem Wissensbasis-Dokument „Preisliste (Richtpreise)“ — niemals erfundene oder frei gerundete Beträge; fehlt die Leistung in der Liste, wird ein Kostenvoranschlag angeboten.
- **Enforced by:** `backend/app/services/agent_config.py:render_price_info_block` (ON branch) + `backend/app/services/price_knowledge.py:render_price_list_text` (doc header: "Nenne NUR Preise, die hier stehen.")
- **Surfaced in UI:** PreisauskunftSection ON-state copy: "Preise, die dort nicht stehen, nennt Kiki nicht."
- **Covered by test:** NONE (pytest); price bucket exists in `backend/tests/agent_evals/scenarios.json`
- **Prompt block:** `# Preise` ({{KZ_PRICE_INFO}})
- **Status:** prompt-only

## PRE-03 — Preisauskunft kann NICHT aktiviert werden, wenn keine aktiven Artikel mit Preis > 0 hinterlegt sind (sonst würde der Agent zu Halluzinationen eingeladen).
- **Enforced by:** `backend/app/api/routes/kiki_zentrale.py` PATCH `/price-info` (`_priced_count` guard → HTTP 422, ~line 1133) and the same guard in the Leitfaden batch save (`save_leitfaden`, returns `no_prices`, ~line 663)
- **Surfaced in UI:** `PreisauskunftSection` — toggle disabled with message "…keine Artikel mit Preisen hinterlegt" (`blockEnable` via `['catalog','price-guard']` query)
- **Covered by test:** NONE found
- **Prompt block:** —
- **Status:** enforced

## PRE-04 — Die Preisliste der Wissensbasis wird automatisch aus den aktiven, bepreisten Katalog-Artikeln erzeugt und bei JEDER Katalogänderung sowie jedem Toggle-Wechsel neu generiert (delete + recreate + re-attach, `usage_mode: auto`).
- **Enforced by:** `backend/app/services/price_knowledge.py:sync_price_list_kb`; triggered as background task in `backend/app/api/routes/catalog.py` (create/update/delete/import, lines 85/171/199/218) and `kiki_zentrale.py` PATCH `/price-info` (line 1161)
- **Surfaced in UI:** PreisauskunftSection (shows priced-position count); Artikel page feeds the catalog
- **Covered by test:** NONE found
- **Status:** enforced

## PRE-05 — Bei deaktivierter Preisauskunft oder leerer Preisliste wird das Wissensbasis-Dokument vom Agenten entfernt und gelöscht (keine veralteten Preise im Agenten).
- **Enforced by:** `backend/app/services/price_knowledge.py:sync_price_list_kb` (disabled/no-items branch → `_detach_and_delete`, clears `agent_configs.price_list_doc_id`)
- **Surfaced in UI:** —
- **Covered by test:** NONE found
- **Prompt block:** —
- **Status:** enforced

## PRE-06 — Ein Wechsel des Preisauskunft-Toggles wird sofort in den Live-Agenten gepusht (Prompt-Repush), damit das Telefonverhalten dem UI-Zustand entspricht.
- **Enforced by:** `backend/app/api/routes/kiki_zentrale.py` PATCH `/price-info` → `_schedule_repush(background, user, "kz_price_info")` (fix for the historical "toggle off but Kiki still quotes prices" bug, see SESSION_HANDOVER.md)
- **Surfaced in UI:** AgentSyncBanner ("Änderungen werden an Kiki übertragen…")
- **Covered by test:** NONE found for the repush wiring
- **Prompt block:** `# Preise`
- **Status:** enforced

## PRE-07 — Im Leitfaden ist „Preisauskunft“ eine VERKNÜPFTE Zeile: ihr Aktiv-Zustand ist zwei-Wege-synchron mit `agent_configs.price_info_enabled` (die Zeile selbst speichert nur die Position), und im Prompt erscheint an dieser Stelle der Verweis auf Abschnitt „# Preise“.
- **Enforced by:** `backend/app/api/routes/kiki_zentrale.py:_LINKED_SETTINGS` + `save_leitfaden`; `backend/app/services/agent_config.py:_LINKED_OFFER_LINES["offer_price_info"]` + `_field_effective_active`; seeded in `backend/app/services/provisioning.py` (offer_price_info row, linked_setting=price_info_enabled)
- **Surfaced in UI:** Kiki-Zentrale → Leitfaden (linked rows warn before toggling, `ConfigSections.tsx` ~line 131)
- **Covered by test:** `backend/tests/test_round2_features.py` (line 343 asserts the seeded field set incl. `offer_price_info`)
- **Prompt block:** Leitfaden offer line referencing `# Preise`
- **Status:** enforced

## PRE-08 — Das Tool hk_queryKnowledgeBase liefert IMMER „keine Informationen“ (Stub) — es ist KEINE Preisquelle; echte Richtpreise kommen ausschließlich aus dem nativen ElevenLabs-KB-Dokument.
- **Enforced by:** `backend/app/services/knowledge.py:query_knowledge_base` (always returns answer=None with German no-info message); route `backend/app/api/routes/tools/query_knowledge_base.py`
- **Surfaced in UI:** —
- **Covered by test:** NONE found
- **Prompt block:** hk_queryKnowledgeBase tool description (template ~line 748 mentions Preise as a query topic, which the stub will never answer)
- **Status:** enforced (note: prompt invites price queries to this tool, but the stub safely returns no-info; the native KB doc answers instead)

## PRE-09 — In KVA-Entwürfen übernimmt der Agent nur im Gespräch konkret genannte Positionen/Preise — er erfindet KEINE Preise.
- **Enforced by:** PROMPT-ONLY — `agent_prompt_template.txt` hk_draftCostEstimate block (~line 762: "erfinde KEINE Preise"); backend `backend/app/services/cost_estimates.py:draft_cost_estimate` normalizes/totals whatever positions arrive but does not validate prices against the catalog
- **Surfaced in UI:** KVA drafts land as status='draft' for team review (Kostenvoranschläge page)
- **Covered by test:** `backend/tests/test_kva_send_and_routes.py` / `test_round2_features.py` cover draft creation mechanics, NOT the no-invented-prices rule
- **Prompt block:** hk_draftCostEstimate tool section
- **Status:** prompt-only (mitigated: drafts require human review at autonomy L1/L2)

- **Enforced by:** `backend/app/services/cost_estimates.py:draft_cost_estimate` (gate on `kva_enabled` with legacy `kva_automation_enabled` fallback → success=False, "KVA-Erstellung ist nicht aktiviert.")
- **Surfaced in UI:** Kiki-Zentrale → KVA-Automatisierung section
- **Covered by test:** `backend/tests/agent_evals/scenarios.json::price_question_toggle_off` (judge_notes assert hk_draftCostEstimate must not fire when off); pytest coverage of the gate not found
- **Prompt block:** hk_draftCostEstimate tool section (~line 756)
- **Status:** enforced
